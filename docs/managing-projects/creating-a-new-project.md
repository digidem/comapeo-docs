## Starting a New Project


In the case where you want to share any observations from My Solo Project, but would rather collaborate with another device on a separate project, then Start a New Project.


> ### 👣 Step by Step  
>   
> _**Step 1:**_ Go to the <img src="/comapeo-docs/images/emojis/three-line-menu_351bd29be717f58f_91463438.png" alt="three-line-menu-black" className="emoji" style={{display: "inline", height: "1.2em", width: "auto", verticalAlign: "text-bottom", margin: "0 0.1em"}} />**Menu**.  
>   
> _**Step 2:**_ Tap on <img src="/comapeo-docs/images/emojis/participant_2dc5f8d1ce7cb4fc_91485326.png" alt="app-icon-participant" className="emoji" style={{display: "inline", height: "1.2em", width: "auto", verticalAlign: "text-bottom", margin: "0 0.1em"}} /> **Collaborate**.  
>   
> _**Step 3:**_ Select <img src="/comapeo-docs/images/emojis/coordinator_77ddbc4137798bed_91485363.png" alt="app-icon-coordinator" className="emoji" style={{display: "inline", height: "1.2em", width: "auto", verticalAlign: "text-bottom", margin: "0 0.1em"}} /> **Start New Project.**  
>   
> _**Step 4:**_ Add a **project name** and continue  
>   
> _**Step 5:**_ Select a **Project Statistics** option. This can be changed in Coordinator Tools  
>   
> _**Step 5:**_ Your new project is ready!

# **Creating a Project**


A **Project** is the workspace where observations, tracks, team information, and settings are stored. Before recording data, you must either join an existing project or create a new one. Creating a project allows you to start documenting information independently or set up a new group workflow.


This page explains how to create a new project on your device, what information is required, and what happens after project creation.


---


## **Accessing the Project Creation Screen**

1. From the main navigation, open the :app-icon-menu: **Menu**.
2. Tap **Projects**.
3. Select **Create a New Project**.

## **Naming Your Project**


You will be asked to enter a project name. Choose a name that clearly reflects the work you will be doing, such as:

- “Forest Patrol – North Zone”
- “River Monitoring Team”
- “Wildlife Survey – July”

To enter your name:

1. Tap the name field
2. Type your project name
3. Tap **Next** or proceed to the next setup step

## **Choosing a Project Icon (If Enabled)**


Some project configurations allow selecting an icon or color to identify the project visually. This helps differentiate multiple projects on the same device.


If available:

1. Tap the project icon selector
2. Choose an icon
3. Confirm your choice

## **Selecting a Project Template (If Available)**


Depending on your configuration, you may see an option to select a **project template**. Templates may include:

- Observation categories
- Custom fields
- Map layers
- Predefined settings

Select the template that fits your work, or choose “Blank” if you want to set everything manually.

## **Finalizing Project Creation**


Once all fields are complete:

1. Tap :app-icon-save-observation: **Create Project**
2. The app will generate your new project
3. You will be automatically taken into the project workspace

Your device is now ready to start recording observations and tracks.

## **Understanding the Project Workspace**


After creating a project, you will see:

- The **Map**, ready for adding observations and tracks
- The **Observations List**, initially empty
- The **Team** section (showing only your device at first)
- The **Menu**, where project settings and export tools appear

## **Joining vs. Creating a Project**


If you join rather than create a project:

- You will see the existing categories, data, and team members
- Syncing or sharing features may distribute data between devices (depending on configuration)

If working independently, creating your own project is the correct choice.


---


## **Tips & Good Practices**


> 💡 <callout icon="💡" color="gray_bg"> Choose a project name that is meaningful to your group. Short but descriptive names are best. </callout>


> 💡 <callout icon="💡" color="gray_bg"> If your project includes multiple teams, consider adding geography or purpose to the name (e.g., “Patrol – East Sector”). </callout>


> ⚠️ <callout icon="⚠️" color="gray_bg"> Changing the name or configuration later may not affect data already collected. Plan your structure before beginning fieldwork. </callout>
