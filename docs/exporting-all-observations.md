# **Exporting All Observations**


Exporting allows you to save all observations on your device into a single file so you can share them or back them up. Exports may include observations, tracks, photos, and other media depending on your project configuration.


This page explains how to export all observations, what the export file contains, and where to find the exported file on your device.


---


## **Why Export Observations?**


Exports are useful for:

- Sharing data with teammates
- Creating offline backups
- Transferring data to computers for analysis
- Archiving project progress

Exports do **not** affect your observations—they remain available in the app.


---


## **Accessing the Export Tool**

1. From the main navigation, open the :app-icon-menu: **Menu**.
2. Select **Export Observations**.

This opens the export screen, where you can review available options.

## **Export Options**


Depending on your project configuration, you may see:

- **Export All Observations**
- **Export Only My Observations**
- **Include Photos** toggle
- **Include Tracks** toggle
- **Export Format** (e.g., ZIP bundle or project-specific formats)

Choose the option that best fits your needs.

## **Exporting All Observations**


To create an export:

1. Select **Export All Observations**
2. Confirm the export in the dialog
3. Wait for the export to complete—progress indicators may appear

Exports usually finish within a few seconds, but may take longer for projects with many photos.

## **Export Completion**


Once the export finishes:

- A message appears confirming the export was successful
- You will see the name of the exported file
- The file is saved to your device’s internal storage

## **Where the Export Is Saved**


The exported file is typically saved in:


```plain text
Device storage / Documents / [Project Name] / Exports
```


Your device may show a shortcut button like:

- **Open Folder**
- **Share File**

This allows you to quickly send the export or access it in a file manager.

## **Sharing the Export**


After exporting, you may choose to share the file directly:

1. Tap :app-icon-share: **Share** if the button appears
2. Select a sharing method (Bluetooth, email, messaging apps, etc.)
3. Send the file to the intended recipient

Exports are standard file types that can be opened on most computers.

## **What’s Included in the Export**


Most exports include:

- All observations
- Tracks
- Photos and media (if enabled)
- Observation metadata
- Category information

This allows project coordinators or analysts to use the data externally.

## **Troubleshooting**


> ⚠️ <callout icon="⚠️" color="gray_bg"> **Export does not start:** Check that your device has enough free storage. Exports that include photos require more space. </callout>


> ⚠️ <callout icon="⚠️" color="gray_bg"> **Export missing photos:** Ensure “Include Photos” was enabled, if your project uses optional inclusion settings. </callout> <callout icon="⚠️" color="gray_bg"> **File not found:** Use your device’s file manager and search for **.zip** files or the project’s export directory. </callout>

## **Tips & Good Practices**


:::note 💡
<callout icon="💡" color="gray_bg"> Export periodically to create backups, especially before major field trips or device changes. </callout>


💡 <callout icon="💡" color="gray_bg"> Label exported files clearly after transferring them to a computer, including date and project name. </callout>
:::
